# BikeStores ASP.NET Core MVC — Design Document

## Problem Statement

Build a **read-only ASP.NET Core MVC web application** for the BikeStores sample database (SQL Server). The application must display product catalogs, store directory, staff, customers, and orders with listing, detail views, and partial-text search — **no CRUD operations allowed**. This is the final exam project for the Business Application Developer course.

## Constraints

- **.NET 10** (SDK 10.0.302, runtime 10.0.10) — confirmed installed
- **SQL Server 2025** — default instance `localhost` with **Windows Authentication** (Integrated Security)
- **BikeStores database** must be created from provided SQL scripts (does not exist yet)
- **No top-level statements** — use `Program.cs` with explicit `Main`
- **No authentication** — public read-only access
- **No CRUD** — only `HttpGet` actions (Index, Details, optional Search)
- **Language: English only** — all UI text, labels, views in English
- **Footer**: "Realizzato da Mario Martins, Corso: Business Application Developer"
- **Solution name**: `BikeStoresProject`, **Project name**: `BikeStores`
- **Delivery**: zip as `Martins_EsameFinale.zip`

## Architecture Overview

Standard **ASP.NET Core MVC** pattern with Entity Framework Core for data access.

```
Browser → HTTP GET → Controller → EF Core (BikeStoresContext) → SQL Server
                          ↓
                      Razor View → HTML → Browser
```

No service layer, no repositories — controllers query EF directly (read-only app, no business logic to warrant abstraction).

## Data Flow

### Listing flow (e.g., Products/Index)
```
GET /Products
  → ProductsController.Index()
  → _context.Products
       .Include(p => p.Brand)
       .Include(p => p.Category)
       .OrderBy(p => p.ProductName)
       .ToList()
  → Products/Index.cshtml renders table
```

### Search flow (e.g., Products/Index?search=trek)
```
GET /Products?search=trek
  → ProductsController.Index("trek")
  → _context.Products
       .Where(p => p.ProductName.Contains("trek")
                || p.Brand.BrandName.Contains("trek")
                || p.Category.CategoryName.Contains("trek"))
       .OrderBy(p => p.ProductName)
       .ToList()
  → Products/Index.cshtml renders filtered table
```

### Detail flow (e.g., Products/Details/5)
```
GET /Products/Details/5
  → ProductsController.Details(5)
  → _context.Products
       .Include(p => p.Brand)
       .Include(p => p.Category)
       .FirstOrDefault(p => p.ProductId == 5)
  → Products/Details.cshtml renders full card
```

## Execution Plan

### Phase 1: Database Setup
1. Run `create objects.sql` on `localhost` (default MSSQLSERVER instance) to create schemas and tables
2. Run `load data.sql` to populate sample data
3. Verify tables exist with `SELECT COUNT(*)` queries

### Phase 2: Project Creation & Scaffold
1. `dotnet new sln --name BikeStoresProject`
2. `dotnet new mvc --name BikeStores --use-program-main --output BikeStores`
3. `dotnet sln add BikeStores/BikeStores.csproj`
4. Add EF Core packages: `Microsoft.EntityFrameworkCore.SqlServer`, `Microsoft.EntityFrameworkCore.Tools`, `Microsoft.EntityFrameworkCore.Design`
5. Scaffold: `dotnet ef dbcontext scaffold "Server=localhost;Database=BikeStores;Integrated Security=True;TrustServerCertificate=True;" Microsoft.EntityFrameworkCore.SqlServer --output-dir Models --context-dir Data --namespace BikeStores.Models --context-namespace BikeStores.Data`
6. Fix `#warning` in scaffolded context
7. Register `BikeStoresContext` in `Program.cs`

### Phase 3: Controller & View Implementation
1. **ProductsController** + Index/Details views + search
2. **StoresController** + Index/Details views + search
3. **StaffController** + Index/Details views + search
4. **CustomersController** + Index/Details views + search
5. **OrdersController** + Index/Details views + search (with order items in detail)
6. **HomeController** (landing page)
7. **Layout** — nav menu, footer, Bootstrap theme

### Phase 4: Polish & Verification
1. Build and verify no errors
2. Run application and manually test each section
3. Verify search functionality for each page
4. Final zip as `Martins_EsameFinale.zip`

## Error Handling

- **Database unavailable**: EF Core throws at runtime — let it propagate as HTTP 500 (no custom error pages required by exam)
- **Entity not found**: Return `NotFound()` HTTP 404 from Details action
- **Empty search results**: Render empty table with "No results found" message
- **Null search term**: Return full list (no filter)

## Testing Strategy

No unit tests — this is an exam project without testing requirements. Validation is manual:
1. Build succeeds (`dotnet build`)
2. Each Index page renders with data
3. Each Detail page renders for valid IDs, returns 404 for invalid
4. Search returns filtered results for partial matches
5. Search returns empty state for no matches
6. Footer text is correct
7. Navigation links work for all sections
